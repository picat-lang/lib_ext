==============================
PICAT COMPACT REFERENCE (v3.9#3)
==============================

========================================
PART 1: LANGUAGE OVERVIEW
========================================

Picat: general-purpose language with pattern-matching, constraints, actors, tabling.
Rules end with "." followed by whitespace/newline.
Two rule types:
  Head, Cond => Body.    (non-backtrackable; commitment, no backtrack)
  Head, Cond ?=> Body.   (backtrackable; backtracks to try next rule if Body fails)
Horn clauses: Head :- Body.   (converted to backtrackable rules)
DCG rules: Head --> Body.     (converted to pattern-matching rules)
Functions: defined with non-backtrackable rules only; head is equation F(...)=X.
Function facts: f(t1,...,tn)=Exp  (shorthand for f(t1,...,tn)=X => X=Exp)
Index declaration (for Horn clauses): index (+,-) (-,+).  + = indexed, - = not indexed.
Private definitions: keyword "private" before definitions hides them from other modules.
Table declaration: "table" before first rule.  Mode-directed: table(+,+,min) etc.

Comments: % to end of line; /* ... */ block comments.

----------------------------------------
STARTUP / COMMAND LINE
----------------------------------------
picat               start interpreter
picat -d            start in debug mode
picat -g Goal       execute Goal instead of main
picat -log          print log/warning info
picat -path Dirs    set PICATPATH (semicolon-separated, double-quoted)
picat -s Size       reserve Size words for stack/heap
picat --help        help
picat --v/--version version
picat File Args...  run File (with .pi optional); main/0 or main/1
                    if args after file: main/1 gets list of strings

Interpreter commands: help, halt/exit (or ctrl-d), cl(File), load(File),
compile(File), cl (from console, end with ctrl-z/ctrl-d).

----------------------------------------
MODULES
----------------------------------------
module Name.          must match filename
import M1,...,Mn.     import modules
include "file".       verbatim include (no module/import in included files)
Default imported modules: basic, io, math, sys.
Module qualifier: M.f(args) calls f from module M.  M must be imported.
Binding order for normal calls: basic,math,io,sys -> enclosing -> imported(in order) -> global.
Binding order for higher-order calls: basic,math,io,sys -> loaded_modules(excl builtins) -> global.
Reserved module names: basic,bp,cp,datetime,glb,io,math,mip,nn,ordset,os,planner,sat,smt,sys,util.

========================================
PART 2: DATA TYPES
========================================

Variables: start with uppercase or underscore. _ is anonymous (each _ is distinct).
  single-assignment.  var(T)/nonvar(T) test free/non-free.
  attr_var(T): attributed variable test.  dvar(T): domain variable test.
  bool_dvar(T): boolean domain variable test.  dvar_or_int(T).
Attributed variables: put_attr(X,Key,Val), put_attr(X,Key) [=put_attr(X,Key,not_a_value)],
  get_attr(X,Key)=Val [throws if missing], get_attr(X,Key,Default)=Val.

Integers: decimal (12_345), binary (0b100), octal (0o73), hex (0xf7). Underscore separators ok.
Reals: 12.345, 0.123, 1.0e8, 12-e10, 0.12E10. Must have . or exponent to be real.

Atoms: lowercase-start identifier, or 'quoted'. Max 1000 chars. \ is escape char.
Strings: lists of single-char atoms. "hello" = [h,e,l,l,o].
Structures: $s(t1,...,tn).  $ distinguishes from function call.  name(S)=name, arity(S)=arity.
Arrays: {t1,...,tn}. Special structure named '{}'. Constant-time access, length stored.
  new_array(D1,...,Dn) creates n-dim array (max 10 dims).
Maps: hash-table of key-value pairs. new_map(S), new_map(N,S), new_map().
  put(Map,Key,Val), put(Map,Key), get(Map,Key)=Val, get(Map,Key,Default)=Val,
  has_key(Map,Key), del(Map,Key), clear(Map), keys(Map)=List, values(Map)=List,
  map_to_list(Map)=Pairs, size(Map)=Size, map(T) test.
Sets: new_set(S), new_set(N,S). Special maps where every key->not_a_value.
Heaps: min-heap or max-heap, represented as arrays.
  new_min_heap(IntOrList), new_max_heap(IntOrList),
  heap_is_empty(H), heap_pop(H)=Elm, heap_push(H,Elm), heap_size(H)=Size,
  heap_to_list(H)=List, heap_top(H)=Elm.

Index notation: X[I] returns Ith component (1-based). X[L..U] returns sublist.
  Multi-dim: X[I,J] = (X[I])[J].  X[I] must be integer or integer range.

List comprehension: [T : E1 in D1, Cond1, ..., En in Dn, Condn]
  T=expression, Ei=iterating pattern, Di=compound value, Condi=optional condition.
  For each tuple of values satisfying conditions, T is added to the list.
Array comprehension: {T : E1 in D1, Cond1, ..., En in Dn, Condn}
  = to_array(list comprehension).

OOP notation: A.f(A2,...,Ak) = f(A,A2,...,Ak) unless A is atom (then module qualifier).
  A.f = A.f() = get(A,f).  Left-associative: X.f().g() = g(f(X)).
  Functions in dot notation always evaluated eagerly.

Ranges: From..To = [From,From+1,...,To].  From..Step..To = [From,From+Step,...,To].

========================================
PART 3: OPERATORS (by precedence, highest to lowest)
========================================

.  @                              (dot, as-pattern)
**                                (right-assoc, power)
unary+  unary-  ~                 (sign, bitwise complement)
*  /  //  /<  />  div  mod  rem    (mult, div, int-div variants)
binary+  binary-                  (add, subtract)
>>  <<                             (right/left shift)
/\                                (bitwise and)
^                                 (bitwise xor)
\/                                (bitwise or)
..                                (range; right-assoc for ++)
++                                (right-assoc, concatenation)
=  !=  :=  ==  !==  =:=  <  =<  <=  >  >=  ::  in  notin  =..
  #=  #!=  #<  #=<  #<=  #>  #>=  @<  @=<  @<=  @>  @>=
#~                                (bitwise not for constraints)
#/\                               (constraint and)
#^                                (constraint xor)
#\/                               (constraint or)
#=>                               (right-assoc, constraint implies)
#<=>                              (constraint iff)
not  once  \+                     (negation, once)
,  &&                             (right-assoc, conjunction)
;  ||                             (right-assoc, disjunction)

Arithmetic: + - * / // /< /> div mod rem ** >> << /\ ^ \/
  /  = real division.  // = truncated integer division.
  /> = ceiling division.  /< = floor division.
  div = floored division.  mod = X - floor(X div Y)*Y.  rem = X - (X//Y)*Y.
  =:=  pretty-much-equal (numerically; 1 =:= 1.0 succeeds, 1 == 1.0 fails).
  =\=  not numerically equal.

Term ordering: var < number < atom < structure/array < list/string.
  @< @=< @<= @> @>=  (term comparison operators)

========================================
PART 5: PREDICATES, FUNCTIONS, PATTERNS, GOALS
========================================

--- Pattern matching ---
Pattern p(t1,...,tn) same form as structure. No function calls, index/dot notation,
  ranges, array/list comprehensions in patterns.
As-patterns: V@Pattern.  V is new var; Pattern is non-var term.
  After matching, V references the matched term.  Avoids re-constructing terms.
Matching: T matches P if P identical to T or P can be made identical by instantiating P's vars.
  Vars in T are NOT instantiated by matching.
  Pred call fails if no rule matches.  Function call throws exception.

--- Goals ---
true, fail/false
p(t1,...,tn)  predicate call
P, Q  conjunction (P && Q is same)
P; Q  disjunction (P || Q is same)
not P  negation; \+ P is same
once P  at most one solution
if (Cond1) Goal1 elseif (Cond2) Goal2 ... else GoalN end
  (elseif/else optional; missing else = else true; if Cond not in parens, must use "then")
throw Exception
!  cut: removes choice points of goals to the left
foreach/while/do-while loops (see Part 6)

--- Tail recursion ---
Last-call optimization: last call reuses stack frame if not protected by choice points.
Accumulator technique for converting to tail recursion.

========================================
PART 6: ASSIGNMENTS AND LOOPS
========================================

--- Assignment ---
LHS := RHS  where LHS is variable or X[...] access.
  X := X+1 compiles to X1=X+1 (new var each assignment).
  Assignment to X[I] updates component; undone on backtrack.

--- Foreach loop ---
foreach (E1 in D1, Cond1, ..., En in Dn, Condn)
    Goal
end
  Ei = iterating pattern, Di = compound value, Condi = optional condition.
  break(Cond) terminates iterator when Cond is true (body NOT executed when Cond true).
  Multiple iterators = nested loops (every combination).
  Variables only in loop = local to each iteration.
  Variables in outer loop = visible in inner loop.

--- While loop ---
while (Cond)
    Goal
end
  Repeats Goal while Cond succeeds.

--- Do-while loop ---
do
    Goal
while (Cond)
  Executes Goal once before testing Cond.

--- List/Array comprehension ---
[T : E1 in D1, Cond1, ..., En in Dn, Condn]
  T=expression, Ei=iterating pattern, Di=compound value, Condi=optional condition.
  For each tuple satisfying conditions, T added to list.
{T : E1 in D1, Cond1, ..., En in Dn, Condn}
  = to_array(list comprehension).

--- Loop scoping ---
Variables only in loop body (not before loop) = local to each iteration.
Variables before loop = global to loop (visible in body).
findall also forms a name scope (local vars not visible outside).
To make a variable global in a loop/comprehension, use it before the loop (e.g., Y=Y).

--- Compilation of loops ---
Loops compiled to tail-recursive predicates.
Variables modified in loop and used after = passed as in/out args to generated predicate.
List comprehensions compiled to foreach loops, then to tail-recursive predicates.

========================================
PART 7: TABLING
========================================

table  (keyword before first rule)
  Tables all calls/answers of predicate/function. Prevents infinite loops and redundancy.
Mode-directed tabling: table(M1,...,Mn)
  + = input, - = output, max = maximize (first best only), min = minimize (first best only),
  mmax = maximize (all best), mmin = minimize (all best), nt = not tabled (last arg only).
  Input args assumed ground; output args assumed variables.
  Only one objective argument allowed (can be compound).
  compare_terms used to compare answers.
initialize_table  clears table area.

========================================
PART 8: CONSTRAINTS (modules cp, sat, smt, mip)
========================================

Import a solver: import cp. / import sat. / import smt. / import mip.
All four modules share same interface on integer-domain variables.
mip also supports real-domain variables (domain = L..U with real L,U).
mip requires Gurobi, CBC, or GLPK. smt requires Z3 or CVC4.
sat also supports bit-vector constraints.

--- Arithmetic constraints ---
Exp1 Rel Exp2  where Rel in: #= #!= #< #=< #<= #> #>=
  #=< and #<= are same (less than or equal)
Allowed in constraint expressions: +,-,*,/,//,count,div,mod,**,abs,min,max,sum
  Also: cond(BoolConstr,ThenExp,ElseExp), count(V,Collection), max(Collection),
  max(Exp1,Exp2), min(Collection), min(Exp1,Exp2), prod(Collection), sum(Collection)
  Index notations, array/list comprehensions interpreted as function calls.
  Dot notation should NOT be used in constraint expressions (evaluated eagerly).
  Constraint in arithmetic expr evaluates to 1 (satisfied) or 0 (not).

--- Boolean constraints ---
#~ BoolExp                     not
BoolExp1 #/\ BoolExp2          and
BoolExp1 #^ BoolExp2           xor
BoolExp1 #\/ BoolExp2          or
BoolExp1 #=> BoolExp2          implies (right-assoc)
BoolExp1 #<=> BoolExp2         iff
BoolExp = 0|1|boolvar|arithconstr|domainconstr|boolconstr
Precedence: #~ > #/\ > #^ > #\/ > #=> > #<=>  (all lower than arith constraints)

========================================
PART 9: ACTORS AND ACTION RULES
========================================

Action rule: Head, Cond, {Event} => Body
  Head = actor pattern, Cond = optional condition, Event = event patterns, Body = action.
  Predicates for actors cannot contain backtrackable rules.
  Conditions must be conjunctions of inline test predicates (type checks, comparisons).
  For each activation, only first applicable rule is applied.
  If action rule applied: actor suspended after body (waits for more events).
  If normal non-backtrackable rule applied: actor vanishes after body.
  Actor fails if no applicable rule or body fails. Body cannot succeed more than once.

Event channels: attributed variables with 4 ports: ins, bound, dom, any.
  Events queued; handled when entering/exiting non-inline predicate/function.
  Inline predicates (X=Y) and functions (X+Y) never interrupted by events.

Event patterns:
  event(X,T)  attach to dom-port of vars in X; T=event term (must be new var)
  ins(X)  attach to ins-port; activated when X instantiated
  bound(X)  attach to bound-port; activated when domain bound changes
  dom(X)  attach to dom-port; activated when inner value excluded from domain
  dom(X,E)  same but E=excluded value (E must be new var)
  dom_any(X)  attach to any-port; activated when any value excluded
  dom_any(X,E)  same but E=excluded value
  atom "generated" in Event: actor suspended AFTER body (not before)

post_event(X,T)  = post_event_dom(X,T)
post_event_ins(X)  post to ins-port of X
post_event_bound(X)  post to bound-port of X
post_event_dom(X,T)  post term T to dom-port of X
post_event_any(X,T)  post event T to any-port of X

freeze(X,Goal)  = once Goal, but delayed until X bound to non-var.
  Defined as: freeze(X,Goal), var(X), {ins(X)} => true.
                freeze(X,Goal) => call(Goal).

Integer: decimal [0-9], hex 0x[0-9a-fA-F], octal 0o[0-7], binary 0b[0-1]
  Underscore separators allowed in numerals.

  Exponent: e/E [sign] decimal
String: " {string_char} "
Comment: /* ... */ or % to end of line

  ^ /\ \/ ~ << >> #= #!= #> #>= #< #<= #=< #=> #<=> #/\ #^ #\/ #~ @> @>= @< @<= @=<

program -> [module_declaration] {import_declaration} program_body
module_declaration -> "module" atom eor
import_declaration -> "import" import_item {"," import_item} eor
program_body -> {predicate_def | function_def | actor_def}
predicate_directive -> "private" | "table" ["(" table_mode {"," table_mode} ")"] |
  "index" index_decl {"," index_decl}
index_declaration -> "(" index_mode {"," index_mode} ")"
index_mode -> "+" | "-"
table_mode -> "+" | "-" | "min" | "max" | "nt" | "mmax" | "mmin"

  head (":-" | "-->") body eor  (Horn clause / DCG)
function_rule -> head "=" expression ["," condition] "=>" body eor
function_fact -> head "=" argument eor
action_rule -> head ["," condition] "," "{" event_pattern "}" "=>" body eor
head -> atom ["(" [term {"," term}] ")"]
condition -> goal
body -> goal
goal -> disjunctive_goal

conjunctive_goal -> conjunctive_goal "," negative_goal | negative_goal
negative_goal -> "not" negative_goal | equiv_constr
equiv_constr -> equiv_constr "#<=>" impl_constr | impl_constr
impl_constr -> impl_constr "#=>" or_constr | or_constr
or_constr -> or_constr "#\/" xor_constr | xor_constr
xor_constr -> xor_constr "#^" and_constr | and_constr
and_constr -> and_constr "#/\" not_constr | not_constr
not_constr -> "#~" not_constr | enclosed_goal
enclosed_goal -> if/foreach/while/do-while | expression {bin_rel_op expression}

  #= #!= #> #>= #< #=< #<= @> @>= @< @<= @=< @>=
expression -> concat_expression
concat_expression -> range_expression ["++" concat_expression]
range_expression -> or_expression [".." or_expression [".." or_expression]]

... (precedence continues through arithmetic and unary to primary_expression)

  float | atom_or_call | list_expr | array_expr | function_call | term_constructor |
  primary_expression "." atom_or_call

  (no loops, if statements, or assignments in term constructors;
   subscript/range/dot/list-comp/array-comp treated as functions in terms)

========================================
PART 24: KEY SYNTACTIC DETAILS
========================================

1. Structures need $ prefix: $f(a,b). Without $, f(a,b) is a function call.
2. Arrays: {a,b,c}. Lists: [a,b,c]. Strings: "abc" = [a,b,c].
3. Function call with 0 args needs parens: f(). Exception: module-qualified (math.pi).
4. Cond in rules cannot succeed more than once (compiler wraps in "once" if needed).
5. Backtrackable rule (?=>): backtracks to try next rule if body fails.
   Non-backtrackable rule (=>): commitment, no backtrack.
6. Cut (!): removes choice points of goals to its left.
7. Assignment := creates new variable at compile time. X:=X+1 becomes X1=X+1.
8. Loop/comprehension variables not used before loop = local to each iteration.
9. findall/find_all forms name scope (like loop): local vars not visible after.
10. Higher-order calls cannot contain assignments or loops.
11. In constraint expressions, expressions are treated as terms (no $ needed),
    EXCEPT index notations, array comprehensions, and list comprehensions which
    are interpreted as function calls. Dot notation should NOT be used in constraints.
12. Horn clauses: Head :- Body. DCG: Head --> Body. Both converted to pattern-matching.
13. index(+,-) declaration for Horn clause predicates: + = indexed, - = not indexed.
14. Private definitions: "private" keyword hides from other modules.
15. OOP: A.f(args) = f(A,args) unless A is atom (then module qualifier).
    A.f = get(A,f).  X.f().g() = g(f(X)).
16. Numerical equality: 1 =:= 1.0 succeeds. Structural: 1 == 1.0 fails.
17. occurs-check disabled in unification (for efficiency). X = f(X) creates cycle.
18. Global maps: max 97, changes NOT undone on backtrack.
    Heap maps: unlimited, changes undone on backtrack.
    Table maps: max 97, hash-consed, changes NOT undone.
19. solve/1 and solve/2 are nondeterministic (can backtrack for multiple solutions).
20. cp allows incremental labeling; sat/mip do not.
21. In mode-directed tabling: + input, - output, max/min (first best), mmax/mmin (all best),
    nt (not tabled, last arg only). Input assumed ground, output assumed variable.
22. Action rule conditions: conjunctions of inline tests only (type checks, comparisons).
    No variable in actor can change during condition execution.
23. Events queued, handled at non-inline call boundaries. Inline calls never interrupted.
24. No primitive to kill/detach actors. Actor vanishes when normal rule applied.
25. freeze(X,Goal): delayed evaluation until X bound.
26. different_terms(T1,T2): disequality constraint, suspended until instantiated.
27. List/array comprehensions: [T : E in D, Cond, ...] - both iterators AND conditions allowed.
28. foreach: break(Cond) terminates iterator when Cond true (body NOT executed).
29. while/do-while: condition tested with Cond (while: before body; do-while: after).
30. As-pattern V@Pattern: V bound to matched term, avoids reconstruction.
31. new_map/new_set: arg can be integer (capacity) or list (initial pairs/keys).
32. sort/1 sorts ascending. sort(List,KeyIndex) sorts by key index.
33. In solve options, $ prefix indicates term (not function call): $max(Var), $min(Var), etc.
34. String is a list of single-character atoms. All list operations work on strings.
35. Array length stored; list length recomputed each call. Array access is O(1).
36. cond(Cond,Exp1,Exp2): conditional expression. = Cond*Exp1+(1-Cond)*Exp2.
37. Functions never fail and never return more than one answer.
38. Predicates can have zero, one, or multiple answers (via backtracking).
39. once(Call): at most one solution. not(Call): negation, never instantiates vars.
40. repeat: always succeeds (repeat ?=> true. repeat => repeat.)

------------------------------
Constraint Programming Advice:
------------------------------

Prefer sat to cp, only try cp if sat seems too slow.

Prefer global constraints to local constraints.

Constraints based on sum in equality and in inequality constraints are powerful, use
this technique (that is very much leveraged in answer set programming).

Prefer 2D or multi-dimensional arrays to list of arrays, if all arrays in
the list have the same length.
